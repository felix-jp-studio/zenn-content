【タイトル】
Firebase×Next.jsで認証機能を30分で実装する5ステップ手順

【note投稿文】

【はじめに】

「ログイン機能ってどうやって作るの？」

HTMLとCSSは書けるようになったけど、いざアプリを作ろうとすると認証機能の壁にぶつかる。そんな経験をしたことはありませんか？

自前でユーザー管理の仕組みを作ろうとすると、データベース設計からパスワードの暗号化まで、初心者には難易度が高い作業が山積みになります。でも実は、FirebaseとNext.jsを組み合わせれば、その壁を驚くほど低くできます。今回は30分で認証機能を実装できる手順を、初心者にもわかりやすく解説します。


【なぜFirebase×Next.jsなのか】

Firebaseは、Googleが提供するバックエンドサービスです。認証・データベース・ホスティングなどの機能がセットになっており、難しいサーバー設定なしに使い始められます。Next.jsはReactをベースにしたフレームワークで、近年のWeb開発で非常に人気が高まっています。

この2つを組み合わせると、以下のメリットがあります。

・バックエンドのコードをほぼ書かずに認証が実現できる
・GoogleやGitHubなどのソーシャルログインも簡単に追加できる
・無料枠の範囲内で個人プロジェクトが十分に動かせる

初心者にとって「動くものを素早く作れる」という体験は、学習のモチベーションを大きく上げてくれます。


【5ステップで実装する手順】

ステップ1：Next.jsプロジェクトを作成する

ターミナルを開き、以下のコマンドを実行してください。

npx create-next-app@latest my-auth-app

プロジェクト作成時の質問はすべてデフォルトのままEnterで進んで問題ありません。

ステップ2：Firebaseプロジェクトを用意する

Firebaseの公式サイト（firebase.google.com）にアクセスし、Googleアカウントでログイン後、新しいプロジェクトを作成します。作成後、「Authentication」メニューから「メール/パスワード」のログインを有効にしてください。

ステップ3：FirebaseをNext.jsに組み込む

ターミナルで以下を実行し、Firebase SDKをインストールします。

npm install firebase

次に、プロジェクト直下にfirebase.jsというファイルを作り、FirebaseコンソールでコピーできるAPIキーなどの設定値を貼り付けて初期化コードを書きます。

ステップ4：サインアップ・ログイン画面を作る

Next.jsのpagesフォルダ内にlogin.jsを作成し、メールアドレスとパスワードを入力するフォームを用意します。Firebaseが提供するcreateUserWithEmailAndPassword（新規登録）とsignInWithEmailAndPassword（ログイン）という関数を呼び出すだけで、認証処理が完成します。

ステップ5：ログイン状態を管理する

onAuthStateChangedという関数を使うと、ユーザーがログインしているかどうかをリアルタイムで検知できます。これをReactのuseStateやuseEffectと組み合わせることで、ログイン済みのユーザーだけが見られるページを作ることができます。


【まとめ】

FirebaseとNext.jsを使えば、認証機能の実装はもはや上級者だけの話ではありません。今回紹介した5つのステップを順番に進めれば、プログラミング初心者でも30分以内に動く認証機能を手に入れられます。「難しそう」と感じていた壁が、実際に手を動かすことで一気に低くなる感覚を、ぜひ体験してみてください。
