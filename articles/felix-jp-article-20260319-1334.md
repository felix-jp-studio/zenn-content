【タイトル】
Firebase×Next.jsで認証機能を30分で実装する5ステップ

【note投稿文】

【はじめに】

「ログイン機能を作りたいけど、どこから始めればいいかわからない」

プログラミングを始めたばかりの方にとって、認証機能の実装はハードルが高く感じますよね。セキュリティの知識も必要そうだし、コード量も多そう……そう思って後回しにしていませんか？

実は、FirebaseとNext.jsを組み合わせれば、HTMLとCSSが書けるレベルの方でも、30分以内にGoogleログイン機能を動かすことができます。今回はその具体的な手順を紹介します。

【そもそもFirebaseって何？】

FirebaseはGoogleが提供するバックエンドサービスです。本来なら自分でサーバーを用意して、ユーザー情報を管理して、セキュリティの設定をして……と大変な作業が必要です。しかしFirebaseを使えば、それらをすべてGoogleのインフラに任せることができます。

認証機能（ログイン・ログアウト）もその1つで、数行のコードを書くだけで安全な認証が実現できます。

【30分で実装する5ステップ】

ステップ1：Firebaseプロジェクトを作成する

Firebaseのコンソール（console.firebase.google.com）にアクセスし、新しいプロジェクトを作成します。プロジェクト名を入力してボタンを数回クリックするだけで完了です。

ステップ2：Authenticationを有効にする

Firebaseコンソールの左メニューから「Authentication」を選び、「始める」をクリックします。次に「Google」を有効化します。ここまで約5分で終わります。

ステップ3：Next.jsプロジェクトにFirebaseをインストールする

ターミナルを開いて以下のコマンドを実行してください。

npx create-next-app my-app
cd my-app
npm install firebase

これでFirebaseをプロジェクトで使う準備が整います。

ステップ4：Firebase設定ファイルを作成する

Firebaseコンソールの「プロジェクトの設定」からアプリを登録すると、設定用のコード（apiKeyなどが含まれるオブジェクト）が表示されます。これをプロジェクト内のfirebase.jsというファイルに貼り付けて保存します。

ステップ5：ログインボタンを実装する

pages/index.jsを開き、以下の処理を追加します。

・GoogleAuthProviderでGoogleログインの設定をする
・signInWithPopupでログインポップアップを表示する
・signOutでログアウトする
・useStateでログイン状態を管理する

ボタンのクリックイベントにこれらの関数を紐付けるだけで、ブラウザ上でGoogleログイン・ログアウトが動作するようになります。

【初心者がつまずきやすいポイント】

・FirebaseのAPIキーをそのままGitHubに公開しないこと（.envファイルで管理する）
・Firebaseコンソールで「承認済みドメイン」にlocalhostを追加しておくこと
・Next.jsのバージョンによってpagesフォルダの構成が異なること

この3点を押さえておくだけで、多くのエラーを未然に防ぐことができます。

【まとめ】

FirebaseとNext.jsを組み合わせることで、認証機能は想像以上に短時間で実装できます。バックエンドの知識がなくても、手順どおりに進めれば今日中に動くものが作れます。

まずは「Googleでログイン」ボタンを1つ動かすことから始めてみてください。それだけで、あなたのアプリは一気にプロらしく見えます。

詳しい手順はこちら → https://felixstudio0.gumroad.com
